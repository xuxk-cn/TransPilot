# TransPilot 普通用户下载版帮助文档

## 1. 这是什么

`TransPilot` 是一个 Windows 截图识别与翻译工具，适合：

- 截图识别图片中的文字
- 把截图内容翻译成目标语言
- 识别表格并导出为 Excel
- 在本地使用模型，尽量不把截图内容上传到外部云服务

## 2. 主要功能

### 2.1 截屏-识图-翻译

使用方式：

- 点击 `截屏-识图-翻译`
- 或按 `Alt+Q`

作用：

- 截图
- 识别图中文字
- 按设置中的目标语言输出翻译结果

### 2.2 表格识别

使用方式：

- 点击 `表格识别`
- 或按 `Alt+E`

作用：

- 截图
- 识别表格
- 导出 Excel

### 2.3 保存图片

在 `截图内容` 区域右侧有 `保存图片` 按钮。

作用：

- 把当前截图保存为 `PNG`

## 3. 下载什么文件

如果你只是普通用户，只需要准备这 3 类内容：

1. `TransPilot.exe`
2. 模型文件
3. `llama.cpp` 运行文件

## 4. 模型文件下载

### 4.1 主模型

直接点击下载：

- [Qwen2.5-VL-7B-Instruct-Q4_K_M.gguf](https://huggingface.co/ggml-org/Qwen2.5-VL-7B-Instruct-GGUF/resolve/main/Qwen2.5-VL-7B-Instruct-Q4_K_M.gguf)

下载后放到：

- `runtime/models/Qwen2.5-VL-7B-Instruct-q4_k_m.gguf`

### 4.2 多模态投影文件

直接点击下载：

- [mmproj-F16.gguf](https://huggingface.co/unsloth/Qwen2.5-VL-7B-Instruct-GGUF/resolve/main/mmproj-F16.gguf)

下载后放到：

- `runtime/models/mmproj-F16.gguf`

## 5. llama.cpp 下载

### 5.1 有 NVIDIA 显卡

直接点击下载：

- [llama.cpp Windows CUDA x64](https://github.com/ggml-org/llama.cpp/releases/download/b8733/llama-b8733-bin-win-cuda-x64.zip)

### 5.2 没有 NVIDIA 显卡

直接点击下载：

- [llama.cpp Windows CPU x64](https://github.com/ggml-org/llama.cpp/releases/download/b8733/llama-b8733-bin-win-cpu-x64.zip)

解压后，把这些文件放到：

- `runtime/llama.cpp/`

至少包括：

- `llama-server.exe`
- 所有同目录下的 `.dll`

## 6. 推荐目录结构

建议整理成这样：

```text
TransPilot/
  TransPilot.exe
  config/
    settings.json
  runtime/
    llama.cpp/
      llama-server.exe
      *.dll
    models/
      Qwen2.5-VL-7B-Instruct-q4_k_m.gguf
      mmproj-F16.gguf
  logs/
  temp/
```

## 7. 如何运行

### 7.1 直接运行

双击：

- `TransPilot.exe`

### 7.2 服务启动逻辑

程序默认会先检查模型服务。

如果没有检测到服务：

- 点击 `检查服务`
- 或直接执行 `截屏-识图-翻译`
- 或直接执行 `表格识别`

程序会尝试启动内置 `llama-server`

## 8. 设置说明

在设置中，普通用户通常只需要关心这几项：

- 模型服务地址
- 界面语言
- 翻译目标语言
- 是否自动启动内置 llama.cpp 服务

默认模型服务地址一般是：

- `http://127.0.0.1:8081`

## 9. 按机器配置如何选模型

### 9.1 低配机器

推荐：

- `Qwen2.5-VL-3B-Instruct` 的 GGUF 量化版本

适合：

- 轻薄本
- 老机器
- 无独显

### 9.2 主流机器

推荐：

- `Qwen2.5-VL-7B-Instruct-q4_k_m.gguf`

适合：

- 有 NVIDIA 显卡
- 日常办公机器
- 追求识图和翻译平衡

## 10. 常见问题

### 10.1 为什么不能只用文本模型

因为这个应用首先要“看懂截图”，纯文本模型不能直接识别图片。

### 10.2 为什么需要 `mmproj`

因为视觉模型在 `llama.cpp` 下通常需要多模态投影文件，没有它图片输入通常不能正常工作。

### 10.3 为什么首次启动会慢

常见原因：

- 模型文件大
- 首次加载模型
- 本机硬件较弱

## 11. 适合谁使用

更适合：

- 需要长期处理外语截图的个人用户
- 不信任云端上传的用户
- 希望本地模型可控的用户
