# TransPilot 管理员部署版帮助文档

## 1. 适用对象

这份文档适合：

- 公司 IT 管理员
- 内网服务部署人员
- GPU 服务器维护人员
- 需要给多人统一提供截图翻译服务的团队

## 2. 部署目标

推荐部署目标：

- 在公司服务器上部署 `llama.cpp`
- 使用统一模型版本
- 多人共享模型服务
- 客户端只负责截图和显示结果

适合场景：

- 外贸
- 制造业
- 海外资料阅读
- 招投标 / 预算表识别
- 内网环境
- 数据敏感部门

## 3. 推荐架构

### 3.1 单机本地版

适合：

- 少量用户
- 无服务器

结构：

```text
TransPilot/
  TransPilot.exe
  config/
  runtime/
    llama.cpp/
    models/
```

### 3.2 公司共享版

适合：

- 多人使用
- 需要统一维护

结构：

- 服务器部署 `llama-server`
- 客户端 `TransPilot` 只连接服务器地址

例如客户端设置：

- `http://10.10.10.20:8081`

## 4. 服务器需要准备什么

至少准备：

1. `llama.cpp`
2. 视觉模型 GGUF
3. `mmproj`
4. GPU 或较强 CPU

## 6. Windows 服务器部署示例

假设目录：

```text
D:\TransPilot\llama.cpp
D:\TransPilot\models\Qwen2.5-VL-7B-Instruct-q4_k_m.gguf
D:\TransPilot\models\mmproj-F16.gguf
```

启动命令：

```bat
@echo off
chcp 65001 >nul
cd /d D:\TransPilot\llama.cpp
llama-server.exe ^
  -m "D:\TransPilot\models\Qwen2.5-VL-7B-Instruct-q4_k_m.gguf" ^
  --mmproj "D:\TransPilot\models\mmproj-F16.gguf" ^
  -c 16384 ^
  -np 4 ^
  -ngl 25 ^
  -t 8 ^
  --port 8081 ^
  --host 0.0.0.0
```

## 7. Linux 服务器部署示例

```bash
./llama-server \
  -m /opt/models/Qwen2.5-VL-7B-Instruct-q4_k_m.gguf \
  --mmproj /opt/models/mmproj-F16.gguf \
  -c 16384 \
  -np 4 \
  -ngl 25 \
  -t 8 \
  --port 8081 \
  --host 0.0.0.0
```

## 8. 参数说明

- `-m`：主模型
- `--mmproj`：视觉投影文件
- `-c`：总上下文
- `-np`：并发槽位
- `-ngl`：GPU 卸载层数
- `-t`：线程数
- `--port`：监听端口
- `--host 0.0.0.0`：允许局域网访问

## 9. 公司多人使用时的建议

`llama-server` 支持并发，但并发能力受：

- 模型大小
- 显存 / 内存
- `-c`
- `-np`
- 图片大小
- 输出长度

### 小团队

推荐：

```bash
-c 16384 -np 4
```

### 中团队

推荐：

```bash
-c 24576 -np 6
```

### 更高并发

建议：

- 不要只靠单实例硬顶
- 采用多实例
- 每实例独立端口
- 前面做分流

## 11. 客户端如何接入公司服务器

客户端在设置中配置：

- 模型服务地址：
  - `http://10.10.10.20:8081`

同时建议：

- 关闭“自动启动内置 llama.cpp 服务”

## 12. 网络与安全建议

- 优先部署在公司内网
- 不建议直接暴露公网
- 控制开放端口
- 保留服务日志
- 限制非授权机器访问

## 13. 运维建议

建议保留：

- 启动日志
- 模型加载日志
- 标准输出 / 错误输出
- 崩溃日志

当前项目默认日志目录：

- `logs/llama-server.log`

## 14. 压测建议

在正式给团队使用前，建议至少做一轮最小压测：

- 5 个并发用户
- 每人连续发 10 次截图翻译
- 再测表格识别

重点观察：

- 平均响应时间
- 峰值响应时间
- 显存占用
- 是否掉线
- 是否出现上下文不足或图片处理失败

## 15. 当前产品卖点

适合作为企业内部工具的原因：

- 截图和文档内容本地处理
- 不需要把业务图片上传到外部云服务
- 可统一部署到公司服务器
- 适合隐私要求高的环境
- 支持截图翻译和表格识别导出 Excel
